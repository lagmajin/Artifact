# マイルストーン: タイムライン左ペインの仮想スクロール化とオーナードロー統一 (2026-04-13)

## 概要
タイムライン左ペインを `QScrollArea` 依存から外し、行本体をオーナードロー + 仮想スクロールで描画する。  
右ペインのタイムライン本体、プレイヘッド、選択ハイライト、スクロール同期を同じ座標系で扱えるようにし、UI の重なりや不整合を減らす。

## 現状の課題
- 左ペインのスクロールが `QScrollArea` に依存しており、右ペインの縦オフセットと完全に同期しづらい。
- 左ペインの選択背景や inline コントロールが、部分的に塗り残しや重なりを起こしやすい。
- 右ペインのプレイヘッドや gizmo オーバーレイと、左ペインの標準ウィジェット構造が別系統で、見た目と操作感がズレやすい。
- 行ごとの描画が QWidget 階層に分散しているため、レイアウト変更や高密度 UI の調整が難しい。
- 左上部の操作帯は要素が詰まりやすく、標準ウィジェットのままだと窮屈に潰れて見えやすい。
- 左ペインで選んだ `Position X` などの property に対して、キーフレームの存在や toggle 入口が見えにくい。
- `Parent` / `Blend` など、レイヤー本体の補助情報と property 編集情報が混ざると視認性が落ちる。

## 方針
- 左ペインの行一覧は `ArtifactLayerPanelWidget` が自前で仮想スクロールし、`paintEvent` で可視行だけ描く。
- スクロール状態は `scrollOffset` として一元管理し、右ペインの縦オフセットと同期する。
- 選択背景、キーフレーム track、inline combo、列区切りは同じ owner-draw キャンバス上で描く。
- 左ペイン上部のヘッダー、検索バー、フィルタ UI も、可能なら owner-draw 寄りにして横幅圧縮に強くする。
- 右ペインとの同期だけが必要な箇所に限って、描画座標とオフセットを合わせる。

## Non-Goals
- QtCSS で見た目を上書きしない。
- Diligent / render backend の描画経路をこのマイルストーンで変更しない。
- タイムライン右ペインのクリップ描画を全面的に作り直さない。

## 実装フェーズ

### フェーズ 1: 左ペインの仮想スクロール化
- `ArtifactLayerPanelWidget` から `QScrollArea` 依存を外す。
- `scrollOffset` と `contentHeight` を内部状態として持つ。
- `wheelEvent` で `scrollOffset` を更新し、`paintEvent` で可視行だけ描画する。
- `scrollToLayer()` はスクロールバー操作ではなく、`scrollOffset` の再計算に置き換える。

### フェーズ 2: 行描画の一元化
- 選択行の背景を row 全体で描く。
- `keyframe track` や inline combo の背景も選択状態と同じキャンバスで描く。
- 行の hover / selected / group row / filtered row の描画規則を明確化する。
- 行の見た目と操作可能領域の境界を揃える。

### フェーズ 2.2: property 行の keyframe 導線
- property 行は `Layer` 行とは別に、編集対象として見分けられる最小限の見た目にする。
- `Position X` などの property 行の右端に、キーフレーム有無を示す小さなマーカーを出す。
- マーカーは現在フレームの keyframe 有無と連動し、クリックで add / remove を切り替える。
- `Parent` / `Blend` のような layer 補助 UI は layer 行に限定し、property 行には出さない。
- `Transform` などの group 行は折りたたみ可能にし、property 行の密度を保つ。

### フェーズ 2.5: 左上部操作帯の圧縮耐性
- 左上部の検索バー、フィルタ、ヘッダー列は、必要最小限の描画要素へ整理する。
- 横幅不足時に文字やボタンが潰れないよう、`paintEvent` ベースの省スペース表現を導入する。
- 列見出しや検索状態表示は、必要ならアイコン化・省略表示・ホバー tooltip 化する。
- 上部操作帯のレイアウト崩れを、標準ウィジェットの auto layout に依存しすぎず抑える。

### フェーズ 3: 右ペインとの同期
- 右ペインの縦オフセット変更を左ペインに反映する。
- 必要なら左ペインのスクロール操作を右ペインにも mirror する。
- 上下方向のスクロール同期が常に片方向・双方向のどちらで動くかを明示する。

### フェーズ 4: 入力とオーバーレイの整理
- プレイヘッド hit-test の死角をなくす。
- gizmo / playhead / overlay が左ペインの重ね描きに影響しないようにする。
- 中ボタン panning、wheel スクロール、ドラッグ編集のイベント経路を整理する。

### フェーズ 5: 回帰確認
- 選択行の背景が欠けないことを確認する。
- 右ペインでのスクロールに左ペインが追従することを確認する。
- プレイヘッドの不感帯が消えていることを確認する。
- レイヤー名検索、inline 編集、キーフレーム track が既存操作と衝突しないことを確認する。
- property 行の keyframe マーカーが、選択/非選択どちらでも読めることを確認する。
- `Parent` / `Blend` / `Physics` など、左ペインで見せない項目が再混入していないことを確認する。

## 完了定義
- 左ペインが `QScrollArea` に依存せず、自前のオーナードローで安定描画できる。
- 右ペインと左ペインの縦同期が自然に保たれる。
- 選択背景の塗り残しや、行コンテンツとの重なり不具合がなくなる。
- プレイヘッドの操作領域がワークエリア周辺でも途切れない。
- 左上部の操作帯も潰れにくく、狭い幅でも状態が読み取れる。
- 上部ヘッダーや検索バーを owner-draw 化しても、UI 全体が過剰に込み合わない。
- property 行から現在フレームの keyframe を付け外しできる。
- layer 補助 UI と property 編集 UI の役割が左ペインで混ざらない。

## Notes
- まずは「行一覧の仮想スクロール」と「右ペイン縦同期」を固める。
- 左上部の検索バーやヘッダーは、詰まり対策として同じ系列でまとめて整理する。
- もし実装リスクが高ければ、上部操作帯だけ独立フェーズに切り出してもよい。
