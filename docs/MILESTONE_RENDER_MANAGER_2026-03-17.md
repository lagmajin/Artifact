# M14 Render Manager (2026-03-17)

レンダーマネージャーを「単なるリスト表示」から、DCC ツールとして実用的な「書き出しワークフローの中核」へ進化させるためのマイルストーン。

## 概要
ユーザーが編集したコンポジションを、高品質な動画や連番画像として、バックグラウンドで効率的に書き出せるようにする。DaVinci Resolve や After Effects のような、プレビューと並行して管理できる UI と、FFmpeg を活用した柔軟な出力パイプラインを構築する。

---

## M-RENDER-1 UI Overhaul & UX Refinement (完了)
- **目標**: 現代的な DCC ツールに相応しい、使いやすく視認性の高い UI 構造を実現する。
- **内容**:
  - `QSplitter` を使用した 2 ペイン（ジョブリスト / 詳細設定）レイアウトの実装。
  - レンダリング詳細設定のグループ化（出力、範囲、オーバーレイ）。
  - リスト操作ボタン（順序変更、複製）のコンパクト化とアイコン化。
  - 中央メインタブエリア（Composition Viewer と同じエリア）への配置変更。
- **完了条件**:
  - ジョブを選択しながら、画面遷移なしで詳細設定を変更できる。
  - 中央タブでのドッキングが正しく機能し、レイアウトが崩れない。

## M-RENDER-2 Async Pipeline & FFmpeg Integration (完了)
- **目標**: UI をフリーズさせず、かつ多様なフォーマットで動画を出力できる基盤を構築する。
- **内容**:
  - レンダリングループのバックグラウンドスレッド化（非同期実行）。
  - `std::thread` による UI スレッドの解放。
  - FFmpeg（QProcess パイプ方式）による MP4/MOV 動画出力の実装。
  - 拡張子による「動画出力」と「連番画像出力」の自動切り替え。
  - 並列レンダリング（TBB）を維持したまま、正しい順序でパイプに書き込む順序制御バッファの実装。
- **完了条件**:
  - 動画書き出し中にタイムライン操作や他設定の変更が可能。
  - 指定したパスに実際に動画ファイル（.mp4）または連番画像（.png）が生成される。

## M-RENDER-3 Advanced Presets & Queue Management
- **目標**: 繰り返し行う書き出し設定を効率化し、複雑なキュー操作を安定させる。
- **内容**:
  - 書き出しプリセット（H.264 High, ProRes 422, PNG Sequence 等）の保存・読込機能。
  - プリセットマネージャの導入。
  - コンポジションの変更（名前変更、削除）に対するキューの追従。
  - プロジェクト保存時のキュー状態の永続化。
- **完了条件**:
  - プリセットを選択するだけで、解像度・コーデック・ビットレートが瞬時に設定される。
  - アプリ再起動後もキューの内容が復元される。

## M-RENDER-4 Real Rendering Integration
- **目標**: 現在の「ダミーレンダリング（テキスト合成）」から「実際のコンポジション描画」への差し替え。
- **内容**:
  - `SoftwareCompositor` を使用した、実際のレイヤー合成結果の書き出し。
  - レンダリング時の品質プリセット（Draft/Final）の適用。
  - オーディオミキサーの結果を FFmpeg パイプに流し込み、映像と音声を多重化（Muxing）。
- **完了条件**:
  - ビューアーで見ている内容と同じものが、指定解像度でファイルとして出力される。
  - 音声付きの動画ファイルが生成される。

## M-RENDER-5 Diagnostics & Monitoring
- **目標**: レンダリングの進捗監視とエラー発生時の原因特定を容易にする。
- **内容**:
  - 詳細なレンダリングログ（各フレームの処理時間、メモリ使用量等）。
  - 書き出し完了時のシステム通知。
  - エラー発生時の詳細な診断メッセージ表示。
- **完了条件**:
  - 失敗したジョブの原因（ディスクフル、不正なパス等）が UI 上で明示される。

## M-RENDER-6 Queue UX, Recovery & Bulk Edit
- **目標**: レンダーキューを、長時間運用でも扱いやすく、失敗時に立て直しやすい作業単位へ引き上げる。
- **内容**:
  - 進捗表示の強化と残り時間推定の精度改善。
  - キュー並び替えの操作性改善（ドラッグ以外の操作、キーボード/ボタン補助の追加）。
  - エラーメッセージの原因可視化（何が失敗したか、どのフレームで落ちたか、どのジョブに紐づくか）。
  - バックグラウンドレンダーの実効性改善。
  - 複数マシンへの分散レンダリングを標準運用に近づける設計整理。
  - レンダー完了後の自動アクション追加（シャットダウン以外の選択肢）。
  - キュー追加時にコンポジションの in/out と work area を確実に反映する。
  - レンダー履歴とログをセッションまたはファイルとして残す。
  - 長時間レンダー中のクラッシュ後に最初からやり直さないためのチェックポイント / 再開機能。
  - 複数アイテムへの一括設定変更（例: 選択ジョブのコーデック一括変更）。
- **完了条件**:
  - 残り時間が「推定値」として妥当な精度で更新される。
  - レンダー失敗時に失敗フレームと原因が UI から追跡できる。
  - 中断後にチェックポイントから再開できる。
  - 複数ジョブの設定をまとめて変更できる。
  - 履歴やログが再起動後にも参照できる。
