# マイルストーン: タイムラインヘッダーのオーナードロー化 (2026-03-23)

## 概要
タイムラインのレイヤーパネル上部にあるスイッチ列（表示、ロック、ソロ等）の見出しヘッダーを、現在の物理的な `QPushButton` 群から、独自の `paintEvent` を用いた「オーナードロー」方式へ移行する。これにより、AE（After Effects）に近い高度なデザイン制御と操作感を実現する。

## 現状の課題
- `QPushButton` を `QHBoxLayout` で並べる方式では、下のレイヤーリスト（オーナードロー）とのピクセル単位の整列が困難。
- ボタンのホバー状態やクリック時のデザインがOS/Qt標準に引っ張られ、DCCツールとしての統一感を損なう。
- 大量のレイヤーがある場合、スイッチボタンのオブジェクト管理コストが発生する可能性がある。

## 移行のメリット
- **デザインの完全制御**: アイコンの配置、フォント、背景グラデーションなどをミリ単位でAE風に調整可能。
- **グリッドの完全同期**: レイヤーリストの列幅（`kLayerColumnWidth`）と完全に一致した描画が可能。
- **柔軟な操作**: ドラッグによる列幅変更や、ヘッダー上での特殊なマウス操作（右クリックメニュー等）の実装が容易。
- **軽量化**: QWidgetオブジェクトの数を減らし、描画のみで完結させることで効率化。

## 実装プラン

### フェーズ1: 構造の再定義
- `ArtifactLayerPanelHeaderWidget` の `Impl` から `QPushButton` メンバを削除。
- `QHBoxLayout` による配置を廃止。
- アイコン画像（QPixmap）を `Impl` で集中管理する仕組みを維持・強化。

### フェーズ2: 描画の実装 (`paintEvent`)
- `paintEvent` をオーバーライド。
- 背景描画: 濃いグレーの背景と、セパレーター（縦線）を描画。
- 列ごとのアイコン描画:
    - 表示（目）、ロック（鍵）、ソロ、サウンド、シャイのアイコンを、対応する列の中央に描画。
- ホバー状態の描画: マウスが乗っている列をわずかに明るくハイライト。

### フェーズ3: インタラクションの実装
- `mousePressEvent`: クリックされた座標から列（インデックス）を判定し、既存のシグナル（`shyToggled` 等）を発行。
- `mouseMoveEvent`: ホバー状態の更新。
- ツールチップの実装: 描画ベースのヒットテストにより、各列に適切なヘルプテキストを表示。

### フェーズ4: 拡張機能（オプション）
- ヘッダー上での右クリックによる「表示項目のカスタマイズ」メニュー。
- 列のドラッグによる並び替え（AE風のスイッチ/モード切替）。

## 完了定義 (Definition of Done)
- ヘッダーの見た目が現在のボタン方式と遜色なく（あるいはそれ以上に）綺麗に描画されている。
- 各列をクリックした際の機能（表示切り替え、ソロ切り替え等）が正しく動作する。
- 下のレイヤーリストの列とヘッダーの列が寸分違わず整列している。
- スペースキーによるパン操作やホイールスクロール中も描画が乱れない。
