# M15 App UX & Core Refinement (2026-03-17)

これまでの実装で判明した「描画アーティファクトの根本原因（QAbstractScrollArea の限界）」や、「レンダーキュー・プロパティUIの進化」を踏まえ、アプリケーション全体の実用性と安定性をさらに一段階引き上げるための改善マイルストーン。

## 概要
DCCツールとして不可欠な「滑らかで安定したUI操作」と「確実に結果が出力されるレンダリングパイプライン」を完成させる。特に、フローティングウィンドウ時のリサイズ描画問題に対する完全な終止符を打ち、キーフレームアニメーションの基礎を結合する。

---

## M-REFINE-1 UI Rendering Stability (脱 QAbstractScrollArea)
- **目標**: Windows 環境下でのドックのリサイズ、フローティング時の描画崩れ（黒い隙間や残像）を完全に撲滅する。
- **内容**:
  - `ArtifactProjectView` で実証された「`QWidget` 継承 ＋ 自前スクロール描画（`painter.translate`）」の仕組みを、他のスクロール領域にも適用する。
  - **対象候補**: `ArtifactTimelineWidget`（タイムライン本体）、`ArtifactInspectorWidget`（プロパティリスト）。
  - 各ビューにおける `QScrollBar` のオーバーレイ化と、リサイズ時の同期描画を徹底。
- **完了条件**:
  - タイムラインやインスペクタをフローティング状態にし、激しくリサイズしても背景が欠けず、内容が即座に追従する。

## M-REFINE-2 Keyframe Animation Integration
- **目標**: DaVinci Resolve 風に刷新されたプロパティエディタのUI（◆ボタン）と、内部のアニメーションエンジンを接続する。
- **内容**:
  - プロパティ名の横にある「◆（キーフレーム）」ボタンを押した際、現在のタイムラインの再生ヘッド位置にキーフレームデータを追加/削除するロジックの実装。
  - 値が変更されたとき、もし現在の時間にキーフレームが存在すれば上書きし、存在しなければ自動キーフレーム（オートキー）のルールに従う。
  - キーフレームが打たれているプロパティの「◆」ボタンを赤色（Checked）にハイライトする状態同期。
- **完了条件**:
  - UIからキーフレームを打ち、時間を進めて別の値を設定すると、その間が補間されてアニメーションする。

## M-REFINE-3 Production Rendering Connection
- **目標**: FFmpeg パイプ出力機能に、実際のコンポジションのレンダリング結果（映像・音声）を流し込む。
- **内容**:
  - `ArtifactRenderQueueService::startAllJobs` 内のダミー描画（`renderSingleFrameDummy`）を廃止。
  - `ArtifactSoftwareCompositor`（または `ArtifactIRenderer`）を呼び出し、実際のレイヤー構成に基づいたフレーム画像（`QImage` / `ImageF32x4_RGBA`）を生成して FFmpeg パイプへ渡す。
  - （段階的拡張）オーディオミキサーからの波形データを抽出し、FFmpeg に音声ストリームとして合成する。
- **完了条件**:
  - プレビュー画面で見ている通りの映像が、動画ファイル（MP4等）または連番画像として出力される。

## M-REFINE-4 UX Polish & Responsiveness
- **目標**: プロフェッショナルなクリエイターが「触っていて気持ちいい」と感じるレベルまで操作感を磨き上げる。
- **内容**:
  - プロパティスライダーの高精度化（10000分解能化）の他ウィジェットへの波及。
  - 値のスクラブ操作中（ドラッグ中）に、過剰な再描画イベントがスタックしないような間引き（スロットル）処理または非同期描画の導入。
  - 各種ドック間のフォーカス移動や、ショートカットキー（Spaceでの再生など）のグローバルな安定化。
- **完了条件**:
  - マウスでのドラッグ操作に対して、画面の更新が滑らかに（60fpsに近い感覚で）追従する。
