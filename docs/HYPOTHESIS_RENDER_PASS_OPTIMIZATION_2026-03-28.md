# Render Pass Optimization Hypotheses (2026-03-28)

現在の `CompositionRenderController` におけるレンダリング負荷を軽減し、特に高解像度・多レイヤー環境でのインタラクティブ性を向上させるための最適化仮説。

---

## 1. レイヤー単位の Dirty 追跡によるスキップ (Granular Change Tracking)

### 【仮説】
レイヤーのプロパティ（色、ソース、エフェクト等）に変更がなく、かつ再生ヘッドが移動していない場合、そのレイヤーのテクスチャ生成パス（CPU Rasterization / GPU Upload）を完全にスキップできる。

### 【詳細】
- **Dirty Flag の分離**: トランスフォーム（位置・回転・スケール）の変更と、コンテンツ（ピクセルデータ）の変更を分ける。
- **キャッシュの永続化**: `LayerSurfaceCacheEntry` をフレームを跨いで維持し、コンテンツが Dirty でない限り再利用する。
- **期待される効果**: 1つのレイヤーを動かしている最中に、他の静止している 100 個のレイヤーの再レンダリングコストがゼロになる。

---

## 2. レンダーパスの統合 (Render Pass Merging / Batching)

### 【仮説】
単純な合成（Normal Blend かつ マスクなし）を行う複数のレイヤーを、個別のドローコールに分けず、1 つの頂点バッファまたはコンピュートシェーダーパスで一括処理できる。

### 【詳細】
- **バッチング条件**: マスク、特殊ブレンドモード、複雑なエフェクトを含まない連続したレイヤーを「バッチ」としてグループ化。
- **テクスチャ・アトラス**: 複数のレイヤー画像を 1 つの大きな GPU テクスチャにまとめ、ドローコールを削減。
- **期待される効果**: レイヤー数に比例して増大する GPU ステートの切り替えオーバーヘッドを削減し、ドローコールを数分の一に抑える。

---

## 3. RenderController でのディスパッチ最適化

### 【仮説】
レンダリングループの極めて早い段階で、描画結果に寄与しない計算（ビューポート外のレイヤー、完全に不透明なレイヤーに隠されたレイヤーなど）を切り捨てることができる。

### 【詳細】
- **ビューポート・クリッピング**: レイヤーの AABB（Axis-Aligned Bounding Box）を計算し、現在のズーム・パン範囲外であれば処理をスキップ。
- **Occlusion Culling**: 前面にある巨大な不透明レイヤー（背景など）によって完全に隠されている背面レイヤーを特定し、描画をスキップ。
- **期待される効果**: 巨大なコンポジションの一部をズームアップして編集している際の負荷を劇的に軽減。

---

## 4. プロパティ反映パスの最小化

### 【仮説】
`ArtifactPropertyWidget` の `rebuildUI`（UIの全構築）を避け、変更されたプロパティ値だけを部分的に反映させることで、UI スレッドのブロッキングを最小化できる。

### 【詳細】
- **Partial Sync**: モデル（Layer）側の変更を検知し、対応する `RowWidget` の数値だけを更新する。
- **期待される効果**: 数値ドラッグ中の UI の「カクつき」を解消し、スムーズな操作感を実現。
